## 🎯 Objetivo da Atividade

Esta atividade tem como objetivo desenvolver um **protótipo de baixa fidelidade** do aplicativo **ReCiclo**, utilizando o **Miro**, com foco na estrutura das telas, no fluxo de navegação e na organização das informações.

A proposta é representar de forma simples a experiência do usuário, priorizando:

- 🧭 Fluxo de navegação
- 🗂️ Arquitetura da informação
- 📱 Estrutura das interfaces
- 👤 Jornada do usuário

Sem se preocupar com o design visual final, cores elaboradas ou elementos gráficos refinados.

---

## 💡 Proposta de Valor

O **ReCiclo** é um aplicativo que conecta pessoas que desejam **doar objetos usados** com pessoas ou instituições que **precisam desses itens**, promovendo sustentabilidade, reaproveitamento e impacto social positivo.

### Principal benefício para o usuário:
O principal benefício do ReCiclo é facilitar o processo de doação e recebimento de objetos de forma simples, rápida e acessível, incentivando a economia circular e reduzindo o desperdício.

---

## 🌍 Sobre o Problema

Uma ONG focada em sustentabilidade deseja lançar o aplicativo **ReCiclo** para conectar:

- **Doadores**, que possuem itens para doar;
- **Interessados**, que buscam objetos disponíveis próximos à sua localização.

O aplicativo permite:

- 📸 Cadastro de itens com foto, descrição e categoria;
- 🔎 Exploração de itens disponíveis;
- 📍 Busca por proximidade;
- 🧰 Aplicação de filtros;
- 💬 Conversa via chat entre usuários;
- 🏅 Gamificação com conquista do **Selo Eco** após doações bem-sucedidas.

## 🖼️ Descrição das Telas

O protótipo apresenta as seguintes telas principais:

### 1. 🔐 Tela de Login
Essa é a tela inicial do aplicativo.  
O usuário informa **username** e **password** para acessar a plataforma.

**Objetivo da tela:**
- Permitir o acesso do usuário ao aplicativo.

**Elementos principais:**
- Campo de usuário
- Campo de senha
- Botão de login

---

### 2. 🏠 Tela de Feed / Home
Exibe os itens disponíveis para doação em formato de lista.  
Cada item apresenta uma imagem ilustrativa, nome do objeto e status de disponibilidade.

**Objetivo da tela:**
- Permitir que o usuário explore os itens disponíveis.

**Elementos principais:**
- Lista de itens
- Imagens representadas por wireframes
- Títulos dos objetos
- Acesso aos filtros

**Exemplos exibidos no protótipo:**
- Coleção de Livro Antigo
- Sofá Usado
- Kit de Roupas

---

### 3. 🧰 Tela de Filtros
Permite refinar a busca por objetos com base em critérios específicos.

**Objetivo da tela:**
- Facilitar a localização de itens de acordo com a necessidade do usuário.

**Filtros disponíveis:**
- Categoria
- Distância
- Condição / estado de conservação

**Elemento principal:**
- Botão **Aplicar filtros**

---

### 4. 📦 Tela de Detalhes do Item
Apresenta mais informações sobre o item selecionado, incluindo imagem, descrição resumida e dados do doador.

**Objetivo da tela:**
- Permitir ao usuário entender melhor o item antes de entrar em contato.

**Elementos principais:**
- Imagem do produto
- Nome do item
- Descrição
- Informações do doador
- Botão **Entrar em contato**

**Exemplo do protótipo:**
- Item: **Sofá Usado**

---

### 5. ➕ Tela de Cadastro de Novo Item
Tela voltada para o doador cadastrar um novo objeto no aplicativo.

**Objetivo da tela:**
- Permitir que usuários ofereçam itens para doação.

**Campos presentes:**
- Área para adicionar imagem
- Descrição do item
- Categoria
- Condição

**Elemento principal:**
- Botão **Publicar**

---

### 6. 💬 Tela de Chat
Permite a comunicação entre doador e interessado para combinar detalhes da retirada do objeto.

**Objetivo da tela:**
- Facilitar o contato direto entre os usuários.

**Elementos principais:**
- Histórico de mensagens
- Campo para digitar mensagem

**Exemplo de uso:**
- Combinar local, horário e forma de retirada do item.

---

### 7. 👤 Tela de Perfil do Usuário
Apresenta as informações do usuário e suas conquistas dentro do aplicativo.

**Objetivo da tela:**
- Mostrar a identidade do usuário e seu engajamento na plataforma.

**Elementos principais:**
- Foto/ícone de perfil
- Nome do usuário
- Tempo de uso da plataforma
- Conquistas e indicadores

**Conquistas exibidas no protótipo:**
- 🌱 Selo Eco
- ⏳ Tempo de uso
- ❤️ Quantidade de doações

## 🧭 Jornada Principal do Usuário

A jornada principal do usuário no aplicativo pode ser descrita da seguinte forma:

1. **Acessa o aplicativo pela tela de login**  
   O usuário entra com seu nome de usuário e senha.

2. **Visualiza o feed de itens disponíveis**  
   Após o login, o usuário navega pela tela inicial para explorar os objetos disponíveis para doação.

3. **Utiliza filtros de busca**  
   Caso queira refinar os resultados, acessa a tela de filtros e seleciona critérios como categoria, distância e condição do item.

4. **Seleciona um item de interesse**  
   Ao encontrar um objeto relevante, o usuário acessa a tela de detalhes para ver mais informações sobre ele.

5. **Entra em contato com o doador**  
   Por meio do botão de contato, o usuário é direcionado à tela de chat para conversar com o responsável pelo item.

6. **Combina a retirada do objeto**  
   No chat, os usuários alinham os detalhes para a retirada ou entrega.

7. **Conclui a doação com sucesso**  
   Após a conclusão, a ação pode gerar reconhecimento dentro do app, como a conquista do **Selo Eco**.

---

## 🔄 Fluxo de Navegação

O fluxo principal entre as telas funciona da seguinte maneira:

**Login → Feed/Home → Filtros → Feed → Detalhes do Item → Chat**  
ou  
**Login → Feed/Home → Cadastro de Novo Item → Publicar**  
ou  
**Login → Perfil do Usuário**

Esse fluxo demonstra como o usuário pode tanto **buscar um item** quanto **realizar uma doação**, além de acompanhar suas conquistas no perfil.
